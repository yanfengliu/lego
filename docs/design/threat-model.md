# Current implementation threat model

This document describes the executable browser and deterministic parallel-worker candidate lab, protocol, catalog, brick-kernel, deterministic maker population, unprivileged captured-output replay harness, renderer and canonical render-packet foundation, strict LDraw profile, and the companion package's content-addressed artifact store and unsealed test-namespace native run ledger currently in this repository. The broader production broker, provider, evaluator, retention, signing, and acceptance design remains normative in [spec.md](spec.md), but those services do not exist yet and are not credited as mitigations here.

## Assets and security objectives

- Preserve the exact user-authored `BrickDocument`, its pinned truth bundle, stable identities, memberships, transforms, connections, provenance, and undo history.
- Prevent untrusted files, build programs, reports, or future provider data from executing code, forging authority, weakening hard validation, escaping scope, exhausting the browser without a deterministic failure, or silently mutating a document.
- Keep development observation hooks and any future provider credentials out of production browser bundles.
- Preserve retained artifact bytes exactly, detect missing or changed CAS objects before use, and prevent artifact metadata from becoming filesystem authority.
- Distinguish structural validity from visual resemblance and avoid physical, provenance, or acceptance claims that the current evidence cannot support.

## Trust boundaries and attacker capabilities

| Boundary | Attacker-controlled input | Current authority on the trusted side |
| --- | --- | --- |
| LDraw import | Bytes, metadata, identifiers, transforms, counts, provenance claims, and internal references | Bounded parser, protocol schema, catalog truth, graph and collision validators |
| Restricted compiler | `unknown` base/program values and broker-shaped scope objects passed by a caller | Structured-cloned inputs, closed schemas, active truth snapshot, deterministic compiler, scope checks |
| Operation application | Runtime JavaScript values that may bypass TypeScript | Structured clone, per-operation schema validation, canonical payload normalization, stale before-values |
| Deterministic maker | Unknown document, brief, scope, and job values | Bounded descriptor-only data snapshot, closed restricted-text policy, exact base/scope binding, trusted compiler, hard validation, immutable evidence, structural deduplication, and a hash-pinned ranking policy |
| Renderer | Documents and optional external validation reports | Local validation, render admission limits, disposable derived scene |
| Canonical render-packet builder | Inert camera, validation, capture-metadata, and artifact-reference records | Data-only descriptor preflight, exact policy-derived camera reconstruction, closed canonical output, protocol validation, and fixed byte/view budgets |
| Browser UI | User gestures, delayed file reads, numeric edits, context loss | React orchestration over immutable kernel results; no mutable Three.js truth |
| Browser candidate lab | Same-build maker output, verifier receipts, stale worker responses, and prompt text | Parallel source-bound maker and verifier workers; bounded structural parsing; version-, request-, and canonical-digest-bound verification; immutable preview-only state |
| Unprivileged replay harness | Canonical test requests, authority-free deterministic maker-output slots, unsealed capture manifests, and stored or caller-supplied capture wrappers | Closed maker-output schema; bounded data-only snapshots and JSON structure; exact bytes, pins, and candidate checkpoints; generator-free downstream recompilation; independent patch reapplication; explicit `passed`, `audit-only`, and `failed` outcomes |
| Development automation | Same-origin development page and model state | `import.meta.env.DEV` guard plus production-bundle token scan |
| Companion artifact store | Untrusted bytes, `ArtifactRefV1` metadata, and references to retained objects | Explicit absolute store-owned root, internally derived SHA-256 paths, byte and metadata policy bounds, bounded operation admission, file-synced atomic publication, and verify-on-read |
| Companion test run ledger | Data-only run, provider-attempt, candidate-transition, idempotency, cancellation-generation, and artifact-reference records | Test namespace only; descriptor-only bounded admission; pinned policy and limits; strict state and exact-checkpoint recovery; hash-linked canonical JSONL; one OS-exclusive writer; file sync and replay verification |

An attacker may submit malformed, deeply nested, oversized, cyclic, aliased, proxy-wrapped, draft-invalid, or internally inconsistent values. They may choose identifiers such as JavaScript prototype property names, reorder set-like arrays, reuse ports, create dense collision geometry, forge validation reports, claim trusted provenance, race duplicate artifact writes, or tamper with artifact bytes. They do not have an implemented provider, credential proxy, broker signing key, acceptance capability, evaluator seal, per-artifact filesystem path, or network API in the current product. The artifact root is trusted companion configuration and must be owned exclusively by that process identity; a party that can replace arbitrary entries inside that private root remains outside this slice's OS-permission boundary.

## Abuse paths and mitigations

| Abuse path | Impact | Implemented mitigation and evidence |
| --- | --- | --- |
| Executable or unknown provider operation | Code execution or hidden mutation | Closed JSON Schema operation union; no dynamic evaluation; runtime operation validation; protocol and compiler rejection tests |
| Mutate a provider object after validation | Candidate/document TOCTOU | Structured clone before schema evaluation and compilation; successful artifacts are recursively frozen |
| Forge catalog, collision, transform, or validator truth | Silent reinterpretation | Reviewed content manifests are hashed into the active truth snapshot; compiler and operation application require exact active truth and otherwise require migration |
| Escape frozen parts, volume, palette, attachment, or count scope | Unauthorized patch | Independent scope rules, free required boundary ports, surviving patch-added attachments, and pre-append operation/add/remove ceilings |
| Hide new invalidity behind a pre-existing aggregate finding | Invalid patch marked valid | Deterministic full-evidence issue identity, bounded display evidence, incomplete-validator sentinels that can never be grandfathered, and regression tests |
| Exhaust validation with dense geometry or issue floods | Browser or worker denial of service | Collision comparison/finding budgets, validation issue/evidence budgets, deterministic blocking sentinel issues, and schema-valid bounded reports |
| Forge manual, AI, template, or migration provenance in a file | False lineage or authority | Unsealed LDraw claims are validated as syntax but reattributed to content-addressed local `import` provenance |
| Alter a named camera or add irrelevant camera fields while labeling a render canonical | Misleading or incomparable visual evidence | The packet builder reconstructs the exact seven-view camera packet from its pinned bounds and camera policy, compares canonical hashes, emits only whitelisted fields, and rejects geometry drift |
| Execute getters or exhaust cloning through malformed render evidence | Code execution or resource exhaustion at the evidence boundary | Descriptor-only bounded data preflight runs before structured cloning; malformed camera, validation, capture, and packet records fail with typed errors |
| Smuggle unsupported intent, authority, parts, colors, references, or an invalid base into deterministic generation | Unauthorized or falsely characterized candidates | The maker accepts only bounded text-only full-model briefs over an exact globally valid empty base, intersects target volume with scope, rejects ambiguous or unsupported intent, and delegates all operations and scope enforcement to the released compiler |
| Rank a duplicate or hard-invalid candidate, or silently change scoring | Invalid candidate presentation or irreproducible comparison | Only compiler-produced globally hard-valid candidates enter ranking; structural duplicates remain compact typed evidence; score weights, diversity constants, and tie breaks are content-hashed in the returned population |
| Return malformed, cyclic, oversized, stale, wrong-build, or forged maker evidence | UI crash, stale preview, or maker-declared validity | The captured maker and dedicated verifier workers run in parallel from the same captured request; the verifier recomputes the expected population and returns only a version- and request-bound canonical digest; the main thread admits bounded data-only records, verifies exact response, candidate, failure, truth, scope, provenance, rank, and digest contracts, and never accepts a late or stale response |
| Smuggle authority, non-normalized programs, forged hashes, omitted slots, changed policies, or self-authored replay success through a capture | False replay evidence or invalid candidate promotion | The maker boundary forbids candidate, scope, provenance, validity, rank, seal, and replay-level claims; replay pins protocol, generation, compiler, ranking and replay policies, recompiles exact captured programs without invoking the generator, recomputes every candidate checkpoint, independently reapplies every hard-valid patch, and derives the terminal report itself |
| Treat a matching rejection or zero-program capture as positive replay proof | Vacuous self-grading | Matching input rejection remains `audit-only`; positive downstream replay requires a compiled program, a hard-valid candidate, independent verification of every hard-valid patch, and all required checkpoints passing |
| Resolve `constructor` or `toString` as catalog data | Prototype lookup confusion | Private `Map` indexes and canonical-ID checks for parts, colors, and allowlists |
| Load external files, paths, or arbitrary LDraw geometry | Path traversal or unreviewed content | Exact metadata-bearing profile, byte/line/count limits, generated internal subfile names, no filesystem resolution, and rejection of external references |
| Render an enormous or forged document/report | Memory exhaustion or waived overlays | Part/primitive admission budgets; local report recomputation; malformed external reports are detached, schema-checked, and ignored |
| Failed scene replacement destroys the prior view | Availability/data-confidence loss | Replacement is fully derived before the old scene is disposed; failed rebuilds retain the prior projection and surface an alert |
| Late import overwrites newer edits | Data loss | File size checked before `text()`, generation token after the asynchronous read, stale result ignored, and discard confirmation before replacement |
| Development observation API ships to users | Production data exposure | Development-only install guard and a production bundle scan for every global/token name |
| Artifact metadata escapes the configured root | Arbitrary file read/write or overwrite | Operations accept bytes and closed `ArtifactRefV1` metadata only; all object paths derive from lowercase SHA-256, every directory is checked against the owned root, and roots or object directories that traverse symlinks are rejected |
| Partial, concurrent, replaced, or corrupted artifact bytes are trusted | Replay corruption or forged evidence | Intrinsic byte-length validation, a prepare-time SHA-256 rechecked after admitted copying, a 64 MiB per-artifact cap, bounded bytes-in-flight/concurrent/pending admission, synced exclusive temporary files, atomic non-replacing directory publication, idempotent same-byte verification, cleanup after caught failures, bounded reads, exact byte-length checks, SHA-256 verification, and tamper/truncation/concurrency tests |
| Concurrent writers, policy drift, sequence forks, truncation, or linked-file swaps alter a native test run | Split-brain or forged replay history | Per-run OS-exclusive writer file, in-process single flight, exact policy/limit binding, canonical event hashes and previous-hash links, monotonic sequences, single-handle recovery truncation, single-link/fstat checks, fsync, and fail-closed reopen |
| Persistence recovery skips candidate or run stages | Invalid candidate reaches presentation or run claims success | Recovery target is derived from the durable pre-failure checkpoint; candidate recovery returns only to that checkpoint or terminal quarantine; run recovery uses a pinned resume map and preserves cancellation generation; success requires a presented candidate and quiescent owned work |
| Late provider output or append floods mutate state or exhaust pending memory | Stale mutation or denial of service | Older generations and terminal-subject output become immutable diagnostic events without state mutation; count and canonical-byte admission is reserved before queued work; invalid transitions are rejected before artifact I/O |

## Residual risk and required future boundaries

- No external model provider is connected. A future provider adapter must bound bytes and JSON depth before parsing or structured cloning, apply the reviewed `ProviderCapabilities` policy, and never forward locked-region-sensitive diagnostics to the maker. Full diagnostics remain broker-local verifier evidence.
- The deterministic maker remains narrow: empty-base text briefs, four project-authored recipes, 10–40 parts, no references, repairs, model calls, authoritative persistence, or acceptance. The browser lab is same-build local diagnostic integration, not the broker, harness, provider, or independent evaluator boundary. Its wall-time field is an admission floor for a bounded synchronous recipe set, not a preemptive deadline. The returned evidence obeys the declared retained-byte ceiling, but authoritative retention still belongs to the future broker ledger.
- The replay harness currently produces deterministic in-memory, unsealed test captures. It proves a limited captured-output downstream claim only; it does not authenticate the maker, bind evidence to CAS/ledger events, retain a finalized bundle, replay canonical pixels, or claim full/sealed replay. Matching input rejection is audit evidence, not positive proof.
- `apps/companion` currently exposes a local library-level artifact store and an unsealed test-namespace event ledger. It has no HTTP surface, job authority, production signing identity, sealed native manifest, retention/tombstone lifecycle, credential proxy, release verification, acceptance authority, or production namespace. CAS identity and test event integrity grant no production authority, consent, or acceptance capability.
- Artifact payload files are synced before publication, but the artifact store still does not fsync containing directories or reclaim crash-stranded staging directories. The test ledger stages root/run initialization atomically and attempts parent-directory fsync, but some Windows filesystems do not support it; power-loss directory-entry durability is therefore best effort, not guaranteed.
- The writer lease never auto-reaps a stale, malformed, partial, or dead-owner lock because automatic takeover could race a new owner. Such a lock fails closed and requires explicit local inspection and manual recovery.
- Ledger artifact resolution is deliberately restricted to the trusted local CAS interface. It has no network timeout or `AbortSignal`; a broken resolver can keep an append, `close()`, and its writer lease pending indefinitely. Network- or provider-backed resolution is blocked until bounded cancellation semantics exist.
- AI candidate acceptance is intentionally unavailable. Enabling it requires the companion broker, a user-originated one-use scope capability, authoritative event recording, transactional application, and an isolated synthetic acceptance drill.
- The renderer is a visual aid, not a structural oracle. The packet builder binds exact metadata, policies, hashes, and canonical camera geometry, but the current browser capture API has not yet issued a trusted receipt that binds each PNG byte hash to the view, pass, camera, renderer, and capture transaction. Relabelled but otherwise valid caller-supplied artifact references therefore remain a residual risk until capture-to-CAS integration lands. Browser pixel comparison, accessibility interaction, WebGL context recovery, and long-run memory evidence remain required delivery gates for visual claims.
- The LDraw profile is tested against this implementation's exact supported subset. External viewer/tool verification is still required before claiming ecosystem compatibility beyond emitted standard part lines and rigid transforms.
- The catalog and collision model make no guarantee of clutch strength, instruction accessibility, mass, cost, inventory availability, or physical stability.
- The browser persists replay-checked project snapshots in local IndexedDB. That database is neither a credential store nor an authoritative broker ledger, and storage failure handling, corruption recovery, quota behavior, and retained-data controls remain browser delivery concerns.

## Regression and gate anchors

- `npm run schema:check` proves generated public types and standalone validators match the authoritative schema.
- `npm run node:check` imports and exercises protocol, catalog, kernel, generation, and rendering packages under the supported Node runtime.
- `npm test` covers hostile wrappers, canonical determinism, scope, invertibility, collision/issue budgets, provenance reattribution, bounded deterministic populations, ambiguous intent, candidate deduplication/ranking/storage budgets, worker-response and verifier-receipt tampering, stale and superseded worker cancellation, render replacement, exact canonical camera geometry, render-packet view and byte bounds, and file-import races.
- `npm run test:browser` covers four-attempt generation, comparison and selection, same-build digest verification, exact candidate-validation-render hash agreement, seven canonical captures, authoring-document non-mutation, post-ready edit clearing, resize and event-loop responsiveness, console errors, exact IndexedDB reload, undo/redo retention, session-only degradation, compare-and-swap races, corruption quarantine, and version-change cleanup in a real browser.
- `npm run build` also proves development automation tokens are absent from production JavaScript.
- `apps/companion/src/artifact-store.test.ts` covers store ownership, intrinsic byte and policy bounds, bounded admission, queued-input mutation, idempotency, concurrent publication, verified reads, tamper, truncation, missing objects, metadata/path attacks, symlink escape, and caught failed-finalization cleanup.
- `apps/companion/src/run-ledger*.test.ts` covers transition and quiescence rules, checkpoint-bound recovery, monotonic cancellation and hash linkage, idempotency, bounded pending and artifact admission, policy pinning, late diagnostics, cross-process writer exclusion, stale-lock fail-closed behavior, fsync/reopen, truncation recovery, corruption, and hard-link swaps.
- `apps/harness/src/deterministic-maker-run.test.ts` covers canonical capture determinism, unsealed authority labeling, policy-pin and manifest tampering, exact-byte admission, generator-free changed-program replay, independent patch verification, non-vacuous proof, and audit-only input rejection.
- `npm run bom:check` and `npm run notices:check` reconcile exact dependencies, provenance records, and notices.
